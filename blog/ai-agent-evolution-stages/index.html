从"能用"到"可训练"：AI Agent 进化的四个阶段

引言

当我们谈论 AI Agent 时，大多数人还在”能用”的阶段。

但真正的变革，发生在 Agent 变得”可训练”的那一刻。

这篇文章，我想从实践者的角度，聊聊 Agent 进化的四个阶段——以及为什么”可训练性”是下一个战场。

一、四个阶段的定义

阶段对照表

阶段特征用户感知典型案例
能用功能闭环”能完成”ChatGPT、Claude
好用体验优化”愿意用”Notion AI、Cursor
懂你个性化”离不开”OpenClaw、Cursor Rules
可训练自定义行为”专属助手”自定义 Agent

这不是线性进化，而是能力的叠加。每个阶段都是下一阶段的基础。

二、阶段 1：能用—— 基础能力的建立

什么算”能用”？

标准：能完成用户交给的任务，不崩溃、不胡说。

核心能力：

  • 自然语言理解
  • 任务执行
  • 错误处理

问题：

  • 每次对话从零开始
  • 没有上下文记忆
  • 行为不可预测

典型案例：早期的 ChatGPT

你问它问题，它回答。你让它写代码，它写。但下次对话，它忘了你是谁，忘了你刚才说了什么。

三、阶段 2：好用—— 体验的打磨

从”能用”到”好用”，差在哪？

差在降低认知负荷。

关键优化点

维度优化前优化后
输入手打长 prompt模板化、快捷指令
输出长篇大论结构化、可复制
错误报错退出自动重试、降级方案
集成孤立工具工作流嵌入

典型案例：Cursor

  • 内嵌 VS Code，不用切换窗口
  • Tab 键自动补全
  • 上下文自动注入当前文件
  • 错误时给出修复建议

本质：好用 = 减少用户的”思考时间”。

四、阶段 3：懂你—— 记忆与个性化

这是最关键的跃迁

从”通用的好工具”变成”你的专属助手”。

实现路径

会话记忆 → 偏好学习 → 行为预测

技术实现

1. 记忆系统

短期记忆（会话级）
    ↓
长期记忆（用户级）
    ↓
持久化存储（文件/数据库）

2. 偏好提取

从对话中学习：

  • 你喜欢什么风格
  • 你常用的术语
  • 你的工作习惯

3. 上下文感知

  • 知道当前时间、位置
  • 知道你在做什么项目
  • 知道你的历史决策

典型案例：OpenClaw

~/.openclaw/workspace/
├── SOUL.md      # Agent 的身份
├── USER.md      # 用户画像
├── MEMORY.md    # 长期记忆
└── AGENTS.md    # 行为规则

这些文件定义了”我是谁”、“你是谁”、“我们如何协作”。

五、阶段 4：可训练—— 自定义行为

这是真正的变革

“可训练”意味着用户可以定义 Agent 的行为，而不仅仅是使用它。

三个层次的可训练性

层次用户能力实现方式
配置调整参数设置界面
规则定义行为规则文件
技能扩展能力插件/技能系统

OpenClaw 的实现

1. 配置层

{
  "model": "qianfan-code-latest",
  "timeout": 180,
  "thinking": "off"
}

2. 规则层

# AGENTS.md

## 子代理触发规则

- 图片分析 → DoubaoCode
- 代码审查 → DoubaoCode
- 快速问答 → 直接处理

3. 技能层

# SKILL.md

## 触发条件
- 用户提及"知识库"
- 需要查询大量文档

## 执行逻辑
1. 初始化向量数据库
2. 插入文档
3. 语义检索

六、可训练性的核心机制

6.1 反馈闭环

传统 Agent：

用户输入 → Agent 输出 → 结束

可训练 Agent：

用户输入 → Agent 输出 → 用户反馈 → Agent 调整 → 记忆更新

6.2 错误学习系统

## [2026-03-11] 配置错误示例

**错误**：飞书群消息无响应
**原因**：requireMention 默认为 true
**解决**：设置 groups.*.requireMention: false
**预防**：配置时明确指定所有布尔参数

每次错误，都成为下次不犯错的依据。

6.3 知识提炼

日常对话 → daily notes → 分析提炼 → 更新长期记忆

不是所有内容都值得记住。Agent 需要”遗忘机制”——保留精华，丢弃噪音。

七、为什么”可训练”是下一个战场？

7.1 从工具到伙伴

“能用”的工具是锤子。 “好用”的工具是电动锤子。 “懂你”的工具是你定制的电动锤子。 “可训练”的工具是能自己学会怎么锤的机器人。

7.2 网络效应

当用户可以训练自己的 Agent：

  • 每个用户的 Agent 都不同
  • 训练经验可以共享
  • 形成技能市场

类比：WordPress 主题市场 → AI Agent 技能市场

7.3 护城河

技术可以复制，但：

  • 用户的训练数据无法复制
  • 个性化的行为规则无法复制
  • 积累的知识无法复制

真正的护城河是”用户投入的时间”。

八、实践建议

如果你在选型

需求选择
偶尔使用”能用”就够了
每天使用选择”好用”的工具
长期协作必须选”懂你”的
定制化需求只有”可训练”能满足

如果你在开发

短期：把基础能力做好（能用、好用） 中期：建立记忆系统（懂你） 长期：设计可训练机制（技能、规则、反馈闭环）

关键问题：

  1. 用户如何”教”Agent？
  2. Agent 如何”记住”？
  3. 用户如何”纠正”Agent？

结语

从”能用”到”可训练”，不是技术的堆砌，而是人与 AI 关系的演变。

  • 能用：我使用工具
  • 好用：我喜欢这个工具
  • 懂你：这个工具懂我
  • 可训练：这个工具会随我成长

未来，每个人都会有自己的 Agent。而区分”我的 Agent”和”别人的 Agent”的，不是它的基础能力，而是它被训练成什么样。

这才是 AI Agent 的真正战场。
